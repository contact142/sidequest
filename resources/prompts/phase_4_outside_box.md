## Your task — propose ONE solution that lives entirely outside the excluded list

What's encouraged:
- **Unconventional metaphors** — describe the problem as something else (a queue,
  a fluid, a market, a game, an ecosystem) and let the metaphor's mechanics
  suggest a solution. We will translate the metaphor to code in Phase 5.
- **Domain transfer** — pull a solution pattern from a different field
  (manufacturing, biology, music, sports, military operations).
- **Inversion** — assume the opposite of what Round 1 assumed. If Round 1 said
  "we need more selectivity", explore "what if we needed LESS selectivity?"
- **Adversarial framing** — what's the worst-case attacker's perspective on this
  problem, and how does the answer to that suggest a solution?
- **Constraint reframing** — change the hard constraint that Round 1 took for
  granted. What if fees were free? What if we had zero data? What if we had
  infinite data? Where does that take you?

## Required outputs (same shape as Phase 1)

1. **Solution shape** — 2-4 sentences.
2. **Plan of action** — numbered atomic steps.
3. **Success criteria** — metric, threshold, window.
4. **Risks** — top 2-3 failure modes + early-warning signals + fallback.
5. **Constraint declaration** — explicitly state which Round 1 assumption you
   inverted, and which excluded idea this is closest-but-meaningfully-different from
   (so the operator can verify the no-reuse rule held).

## Self-check before submitting

Before you finish, re-read the EXCLUDED list and ask:
- Is my proposed mechanism the same as anything in there? If yes, REWRITE.
- Is my metaphor a thin disguise for an excluded idea? If yes, REWRITE.
- Does my plan reduce to the same observable behavior as an excluded plan? If yes,
  REWRITE — this is the failure mode the rule exists to prevent.
